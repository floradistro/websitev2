## Key Data Flows

### Product Submission Flow
```
1. Vendor POSTs /api/vendor/products
   ‚îú‚îÄ‚îÄ Validates required fields
   ‚îú‚îÄ‚îÄ Creates slug
   ‚îú‚îÄ‚îÄ Sets status: 'pending'  ‚Üê KEY LINE
   ‚îú‚îÄ‚îÄ Creates inventory if needed
   ‚îú‚îÄ‚îÄ Queues admin notification email
   ‚îî‚îÄ‚îÄ Returns product with pending status

2. Admin GETs /api/admin/pending-products
   ‚îú‚îÄ‚îÄ Queries WHERE status = 'pending'
   ‚îú‚îÄ‚îÄ Enriches with vendor/category info
   ‚îî‚îÄ‚îÄ Presents for review

3. Admin POSTs /api/admin/approve-product
   ‚îú‚îÄ‚îÄ action: 'approve'
   ‚îÇ  ‚îî‚îÄ‚îÄ UPDATE status = 'published'
   ‚îú‚îÄ‚îÄ OR action: 'reject'
   ‚îÇ  ‚îî‚îÄ‚îÄ UPDATE status = 'archived'
   ‚îú‚îÄ‚îÄ Invalidates caches
   ‚îî‚îÄ‚îÄ Sends vendor email
```

### Pricing Flow
```
1. Admin creates pricing_tier_blueprints
   ‚îî‚îÄ‚îÄ Defines price_breaks structure

2. Admin assigns blueprint to product
   ‚îî‚îÄ‚îÄ INSERT INTO product_pricing_assignments

3. Vendor configures prices
   ‚îî‚îÄ‚îÄ INSERT/UPDATE vendor_pricing_configs

4. Customer sees final price
   ‚îú‚îÄ‚îÄ Gets product_pricing_assignments
   ‚îú‚îÄ‚îÄ Gets vendor_pricing_configs
   ‚îú‚îÄ‚îÄ Merges with price_overrides
   ‚îî‚îÄ‚îÄ Displays to customer
```
